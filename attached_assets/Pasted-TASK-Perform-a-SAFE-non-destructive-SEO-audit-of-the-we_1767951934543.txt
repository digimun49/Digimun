TASK: Perform a SAFE, non-destructive SEO audit of the website and recommend improvements.

Project context:
- Static HTML website
- Hosted on Netlify
- Clean URLs implemented (no .html visible)
- Firebase used only for authentication
- SEO already implemented (meta tags, structure present)

IMPORTANT RULES:
- DO NOT automatically change content
- DO NOT remove or overwrite existing SEO tags
- DO NOT add keyword stuffing
- DO NOT touch Firebase or backend logic
- Provide recommendations first, then optional patches if explicitly requested

Audit scope:

1) Meta Tags Review
   - Check presence of:
     - <title>
     - meta description
     - viewport
     - charset
   - Identify duplicates across pages
   - Flag missing or overly long titles/descriptions

2) Headings Structure
   - Verify exactly ONE <h1> per page
   - Check logical hierarchy (h1 → h2 → h3)
   - Flag skipped levels or misuse

3) Image SEO
   - Check all <img> tags
   - Identify missing or generic alt attributes
   - Flag large images that may affect performance

4) Internal Linking
   - Verify all internal links use clean URLs
   - Identify orphan pages (no internal links pointing to them)
   - Suggest contextual linking (not sitewide spam)

5) Canonical & Indexing
   - Check if rel="canonical" exists
   - Ensure canonical points to clean URL version
   - Flag pages that should potentially be noindexed (login, signup, admin)

6) Performance Signals (Frontend only)
   - Flag:
     - render-blocking scripts
     - unminified JS/CSS
     - missing defer/async
   - No automatic minification

7) Structured Data (Optional)
   - Check if schema.org markup exists
   - Suggest SAFE schemas only:
     - Organization
     - Website
     - Breadcrumb (if applicable)
   - Do NOT inject schemas automatically

8) Sitemap & Robots (Check only)
   - Verify sitemap.xml existence
   - Check robots.txt rules
   - Flag conflicts or missing references

Output required:
- A concise audit report grouped by:
  HIGH priority (should fix)
  MEDIUM priority (recommended)
  LOW priority (optional)
- For each item:
  - Page(s) affected
  - Reason
  - Safe recommendation
- NO automatic changes unless explicitly approved later
