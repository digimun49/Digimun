1. BAHUT KHATARNAAK (Critical Issues)
A. Admin APIs mein koi real authentication nahi hai
Yeh sabse bada masla hai. Aapke saare admin endpoints (delete-account, signal-delete, signal-edit, batch-approve etc.) sirf yeh check karte hain ke request body mein jo adminEmail bheja gaya hai woh match karta hai ya nahi. Lekin server kabhi verify nahi karta ke request actually admin ne bheja hai ya nahi (Firebase token check nahi ho raha).

Iska matlab: Agar kisi ko aapka admin email pata chal jaye (jo ke already Firestore rules mein muneebg249@gmail.com hardcoded hai), toh woh koi bhi admin action kar sakta hai — users delete karna, signals edit karna, batches approve karna — bina login kiye.

B. check-admin endpoint admin email leak karta hai
Koi bhi insaan /.netlify/functions/check-admin pe koi bhi email bhej ke check kar sakta hai ke admin kaun hai. Ek simple script se woh 100 emails try karke admin email discover kar sakta hai.

C. CORS policy sabke liye open hai
Server aur saari Netlify functions mein Access-Control-Allow-Origin: * set hai — iska matlab koi bhi doosri website se aapke APIs call kar sakta hai.

D. Firestore rules mein admin email exposed hai
muneebg249@gmail.com hardcoded hai — code dekhne wala koi bhi developer admin email jaan jayega.

2. HIGH RISK Issues
E. Premium content protection sirf browser mein hai
access-check.js aur access-gate.js sirf client-side (browser mein) check karte hain ke user ko access dena hai ya nahi. Lekin actual pages (signal.html, DigimunX etc.) toh sab ko serve ho rahi hain. Koi developer DevTools khol ke JavaScript disable kare, ya directly URL call kare, toh premium content access ho jayega.

F. Admin 2FA session localStorage mein store hai
2FA verify hone ke baad, session 3 din ke liye localStorage mein save hota hai. Koi developer DevTools console mein ja ke localStorage.setItem('admin2FAVerified', 'true') likh ke admin panel bypass kar sakta hai. Device fingerprint bhi weak hai.

G. Server pe Firebase Token kabhi verify nahi hota
Koi bhi endpoint user ka Firebase ID token check nahi karta. Matlab koi bhi bina login ke API calls kar sakta hai.

H. Poora project root static serve ho raha hai
express.static('.') — iska matlab saari files (JS code, admin panel HTML, configuration files) publicly accessible hain. Admin panel ka URL /admincontroldp49 sirf obscurity se chhupa hai, security se nahi.

3. MEDIUM RISK Issues
I. Do alag Firebase projects use ho rahe hain
firebase-config.js mein digimun-chat project hai aur firebase.js mein digimun-49. Yeh confusion paida karta hai aur security gaps ka chance barhata hai.

J. Tickets/uploads bina authentication ke ho sakte hain
/api/upload-ticket-attachment pe koi authentication check nahi — koi bhi files upload kar sakta hai.

K. Netlify function loader
Dynamic function loading (loadNetlifyFunction) mein path validation limited hai — haalake regex se kuch protection hai.

4. LOW RISK Issues
L. Zyada tar endpoints pe rate limiting nahi hai
Sirf 2FA pe rate limit hai. Login, signup, signal analysis, file uploads — sab pe unlimited requests bheji ja sakti hain.

M. Error messages mein internal details leak hoti hain
Kuch errors mein server ki internal information show hoti hai jo attacker ke liye useful ho sakti hai.

Kya aap chaahte hain ke main in issues ko fix karun? Main pehle sabse critical fixes (admin authentication, CORS, token verification) se shuru karun ga.
Code review bhi confirm karta hai ke yeh issues serious hain.
